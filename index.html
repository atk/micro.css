This minimal CSS framework is by no means complete. It is meant to be extended, partially replaced or used as a learning example for the following features:

  • Lightweight
  • Responsive
  • Modular
  • Extendable
  • Themeable with CSS variables (requires postCSS or a modern browser)

It is not using a CSS reset or normalize, as that would interfere with normal browser styles and reduce interoperability with other CSS modules.

Fonts

Feel free to add your own fonts, in normal, italic and bold faces as you please. Scale up the window to see the responsiveness of the font inclusion in browsers with support for calc.

Grid

The grid uses flexbox (without flex-wrap) to allow for flexible layouts in modern browsers (Chrome 4+, Firefox2+, MSIE10+, Edge, Opera 12+, Safari 3.1+, Android 2+). It has 3 breakpoints: L 960+, M 640-960 and S 640-.

Other browsers will show the content one cell / box below another, so it's degrading more or less gracefully - the page will not be as beautiful, but it should be still usable. The grid works by using the classes row/col (or prefixed with the respective breakpoint, like l-row) for the grouping and cell or box for the content.

There is also a module that allows similar layouts using display: table(-cell) which uses more markup and is more difficult to use, but is supported even by older browsers.

Images

The following image will fill the width of the relative parent:

The image to the left will scale to full size in mobile viewports and otherwise will retain its original size. It will otherwise float on the left side, displacing content as the browser sees fit. article containers (both with the tag or the WAI ARIA role) will clear the floating, thus allowing you to use floated images without displacing the subsequent content.

The Image on the right will adjust itself to a quarter of the available space in displays bigger than 960px, a third in those bigger than 640px and half below that. The classes left, right, full, half, third and quarter allow for flexible image formatting that will work for all your tags including img, svg, video, object/embed and picture.

Be sure to include an alt-attribute in all your img-tags and may your code always be valid.
